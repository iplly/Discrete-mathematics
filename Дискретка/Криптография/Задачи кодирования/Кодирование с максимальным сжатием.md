### Энтропия ^entropy
Пусть источник порождает символы $a_i$ с вероятностями $p_i$. Энтропия:
$$H = -\sum_{i} p_i \log_2 p_i$$
Нижняя граница средней длины кода: $\overline{L} \geq H$

### Код Шеннона-Фано ^shannon_fano
Алгоритм:
1) Упорядочить символы по убыванию вероятности
2) Разделить на две группы с примерно равной суммой вероятностей
3) Присвоить первой группе «0», второй — «1»
4) Рекурсивно повторить для каждой группы

Средняя длина не превышает $H + 1$, но код не всегда оптимален.

### Код Хаффмана ^huffman
Оптимальный префиксный код. Алгоритм (снизу вверх):
1) Упорядочить символы по вероятности
2) Объединить два наименьших в один узел (сумма вероятностей)
3) Повторять, пока не останется один корень
4) Ветви помечаются «0» и «1»

Префиксное свойство: ни одно кодовое слово не является началом другого.

### Арифметический код ^arithmetic
Кодирует всю последовательность в одно число из $[0,1)$.

**Алгоритм:**
1) Начальный интервал $[0,1)$
2) Для каждого символа $a_i$:
   - Разделить интервал пропорционально вероятностям
   - Выбрать подынтервал, соответствующий $a_i$
3) После обработки всей последовательности передать любое число из финального интервала

Даёт сжатие, близкое к энтропии, даже для символов с вероятностью $\approx 1$ бит.
